Introduction

This document describes the first draft specification of the Tonality Ontology developed in the Centre for Digital Music, Queen Mary, University of London. It has been created as part of the OMRAS2 project and is aimed at providing descriptors for tonal content in RDF.

The ontology is still in the testing stage and it has been only used by Henry to publish key features extracted from Vamp Plugins in RDF format providing additional information of the harmonic expansion of the key. The specification is available as part of the motools project on Sourceforge.

All feedback is welcome and if you would like to propose changes or additions to the ontology, the best place to do so is on the Music Ontology mailing list. If you prefer, please feel free to contact the author of the document.

Acknowledgements

The author would like to thank the Centre for Digital Music and the Music Ontology Community for the necessary support to develop this work. This specification has been implemented in hopes of being a useful tool in the music research community and future discussions and contributions will be included in following versions of the document.

The design and layout of this ontology document is based on the Music Ontology and FOAF Vocabulary specification documents.

Ontology Terms

Classes

Class: to:Degree - unstable -

Degree - Ordered sequences owns pitch relationships among their members. Each member is a degree and its tonal and harmonic properties depend on its position in the sequence.

[back to top]


in-range-of:degree
in-domain-of:borrowed_chord owned_chord frequency pitch pitch_class degree_position degree_name interval

Class: to:Interval - stable -

Interval - Distance in pitch between two different sounds. Intervals can be referred to distinct tuning systems and measured with different metrics. Intervals establish the relationships among pitches, so they are the basic elements to build musical structures. The interval corresponds physically to a frequency distance, but the psychological distance between two pitches is not derived from it.

[back to top]


in-range-of:interval simple_interval enharmonically_equivalent inversion
in-domain-of:strength octaves simple_interval enharmonically_equivalent inversion interval_class just_cents cents tuning comparable_ratio ratio scale_steps semitones steps

Class: to:IntervalClass - unstable -

IntervalClass - Generalization if interval classes representing the between two unordered pitch classes where octave, enharmonic and inversional equivalencies are assumed. There are then 7 classes (0-6) representing distances from the minimum to the maximum value possible within the octave.

[back to top]


in-range-of:interval_class
in-domain-of:semitones steps
sub-class-of:Interval

Class: to:Key - unstable -

Key - Keys are a musical element mainly defined and useful within the western music. It is quite difficult to give a precise definition of key that everybody agrees with. In the Tonality Ontology, the key is seen as higher abstraction of scales only meaningful in traditional tonal music.
Keys own a tonic and a mode (normally major and minor) and are used as pattern for stable compositions. Note that the minor mode of a key is a mixture of the minor mode of the diatonic set and the harmonic and melodic minor scales.
On the other hand, keys are very important in harmony. A chord progression establish a key through a fragment and vice-versa the key establish the harmonic function of a set of chords.

[back to top]


in-domain-of:scale tonic mode degree

Class: to:KeyMode - testing -

KeyMode - Minor or major modes of a key.

[back to top]


sub-class-of:Mode

Class: to:Mode - stable -

Mode - Sets can be ordered in several ways forming different scales. The ways we can order a set are the "Modes", so each scale has a particular shape which really describes it. From a tonic and a mode we can generate a scale.

[back to top]


in-range-of:mode
in-domain-of:degree

Class: to:Pitch - unstable -

Pitch - Pitch is the perception of the fundamental frequency of a sound. Each played note (a real event) owns physical properties that can be viewed also as human perceptions. The matching between physical and psycological properties is not linear either a one-to-one set of relationships. Tonality is about pitches perception and relationships within a meaningful expression. Pitches are represented by symbols (symbolic notes) used to categorize our perception of sounds as a musical vocabulary.

[back to top]


in-range-of:tonic pitch
in-domain-of:frequency octave pitch_class

Class: to:PitchClass - unstable -

PitchClass - Generalization of pitch families where octave, enharmonic and inversion equivalencies are assumed. Equivalent pitches in different octaves are wrapped in the same class. Thus, different frequencies fall in the same musical category.

[back to top]


in-range-of:pitch_class
sub-class-of:Pitch

Class: to:Scale - stable -

Scale - Musical sets can be ordered setting a tonic or a final pitch, forming scales. Then, each member of the set plays a role in the scale in respect of the tonic. Therefore, scales are ordered sequences of pitches in respect of a tonic. A progression of notes along a fragment of a musical work can be also considered a scale with a final note, but in this case those notes belongs to a more elementary structure understood as a scale in the first sense.

[back to top]


in-range-of:scale
in-domain-of:num_degrees tonic mode degree interval

Class: to:Set - unstable -

Set - Music set formed of an unordered bunch of musical entities. For example {C, D, E, F, G, A, B} is the diatonic set for the key C major.

[back to top]


in-domain-of:interval

Class: to:TonalSegment - stable -

TonalSegment - A tonal classifier of one particular segment of the timeline of a signal with specific tonal content defining tonal regions. Each tonal element associated with the event is considered as a factor. The audio features ontology owns a wider range of classifiers related to tonality and other features.

[back to top]


sub-class-of:af:MusicSegment

Properties

Property: to:borrowed_chord - unstable -

borrowed_chord - Some chords owned by a key (infered for the underlying scale) are not very stable so some alterations are normally present to obtain more stability like the V degree of the relative minor key. Some degrees substitute its seventh owned interval by some other seventh interval from the parallel key giving a borrowed chord but still diatonic (altered chords came from non diatonic intervals). An important case is also the alteration of the dominant degree of the minor key to obtain a tritone producing the harmonic minor scale (can be seen as a chord borrowed from the major relative key). Borrowed chords are normally used in a key (minor or major), however other chords can be used while playing in a particular key. These chords should be classified as other tonal events (see audio features ontology).

Domain: Degree
Range: chord:Chord

Property: to:cents - unstable -

cents - Cents measurement: 1200×log2(f2/f1). This value is used to compare the size of intervals.

Domain: Interval
Range: xsd:decimal

Property: to:comparable_ratio - unstable -

comparable_ratio - Closest ratio in a Just Intonation tuning for a given equally-tempered interval. (This is actually very optional).
Domain: Interval
Range:

Property: to:degree - unstable -

degree - Binds a degree resource to a structure.
Domain: Mode Scale Key
Range: Degree

Property: to:degree_name - unstable -

degree_name - Property that expresses the name of the degree which normally will convey implicitly its harmonic functionality. We declare a specific property for this instead of using any other name property.

Domain: Degree
Range:

Property: to:degree_position - unstable -

degree_position - Position of the degree within the scale which will determine its functionality.
Domain: Degree
Range: xsd:integer

Property: to:enharmonically_equivalent - unstable -

enharmonically_equivalent - Two intervals are enharmonically equivalent when they express the same pitch distance but as different general intervals, so they have the same number of semitones but different number of scale steps. Enharmony is a general musical property which states that two different musical elements has the same acoustic value.

Domain: Interval
Range: Interval

Property: to:frequency - unstable -

frequency - Express the real frequency of the pitch.
Domain: Pitch Degree
Range: xsd:decimal

Property: to:interval - unstable -

interval - This property allows to link an interval to some other structure. So far we define intervals for Scale, Degree (which link it to a Scale or a Key) and Chord.

Domain: Set chord:Chord Degree Scale
Range: Interval

Property: to:interval_class - unstable -

interval_class - The Interval Class (0-6) which the interval belongs to according to the semitones property
Domain: Interval
Range: IntervalClass

Property: to:inversion - unstable -

inversion - This property links an interval with its inversion. Major second is the inversion of minor seventh.
Domain: Interval
Range: Interval

Property: to:just_cents - unstable -

just_cents - Just cents does not represent the real cents of the interval but the cents value of the comparable ratio for a equal-tempered interval. This is very optional as well in interval definition.

Domain: Interval
Range: xsd:decimal

Property: to:mode - unstable -

mode - Mode of a scale (its shape). The scale is well defined by a tonic or reference note and a mode (different organizations of a set). Modes are also used to characterize keys, but in a slightly different way. There are only two modes for keys (major and minor) which actually do not wholy correspond to the minor and major scale modes. Both structures can be defined by an ordered sequence of notes.

Domain: Key Scale
Range: Mode

Property: to:num_degrees - unstable -

num_degrees - Number of steps in a scale.
Domain: Scale
Range: xsd:integer

Property: to:octave - unstable -

octave - Specify the concrete pitch in a particular octave. Eg: A in the 3rd octave.
Domain: Pitch
Range: xsd:integer

Property: to:octaves - unstable -

octaves - Number of octaves comprised in an compound interval.
Domain: Interval
Range: xsd:integer

Property: to:owned_chord - unstable -

owned_chord - This property allows to associate chords to each of the degrees of the key describing the harmonic properties of the tonality. We will associate 7th chords as standard, but any chord with the same functionality would be valid as well. Other chords will be then borrowed from a key not actually present or produced by the use of a chromatic interval (to avoid a tritone for example). See borrowed_chord

Domain: Degree
Range: chord:Chord

Property: to:pitch - unstable -

pitch - Associates the degree to a total pitch which is determined by a note in a specific octave. This is only necessary when the tuning system is not equally tempered so pitches can not be easily infered from octave to octave (non-octave equivalency).

Domain: Degree
Range: Pitch

Property: to:pitch_class - unstable -

pitch_class - This property expresses the same information than note except for the difference we consider between note and pitch class. This should be better considered.

Domain: Pitch Degree
Range: PitchClass

Property: to:ratio - unstable -

ratio - Measure of intervals in Just Intonation system as a ratio of the frequencies (simple rationale numbers).
Domain: Interval
Range:

Property: to:scale - unstable -

scale - Binds a scale to other structures.
Domain: Key
Range: Scale

Property: to:scale_steps - unstable -

scale_steps - Steps of the scale comprised in the interval. This is used to characterized general intervals and relate specific intervals to the a certain degree.

Domain: Interval
Range: xsd:integer

Property: to:semitones - unstable -

semitones - Convenience measure in EDO systems. Treated as distance it doesn't represent the real tonal distance but a frequency distance.
Domain: IntervalClass Interval
Range: xsd:integer

Property: to:simple_interval - unstable -

simple_interval - In equal temperaments compound intervals (bigger than a octave) are indeed a close interval and n octaves. This property relates an compound interval like minor ninth with its close version minor second.

Domain: Interval
Range: Interval

Property: to:steps - unstable -

steps - Measure for intervals in equally-tempered systems
Domain: IntervalClass Interval
Range: xsd:decimal

Property: to:strength - unstable -

strength - This property expresses the stability of an interval given its tuning. Although is very difficult to state that a certain interval is stable or not.

Domain: Interval
Range:

Property: to:tonic - unstable -

tonic - Sets the tonic of a key or scale if the first degree is not bound to a note.
Domain: Scale Key
Range: Pitch

Property: to:tuning - unstable -

tuning - The tuning system in which we consider the interval. Through this property we do not restric the interval definition to an equally temperament but to any temperament and division of the octave.

Domain: Interval
Range: tuning:TuningSystem
